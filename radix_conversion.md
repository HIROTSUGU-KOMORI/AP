## _基数変換_

---
### _基数_

我々が普段使用している数字は0~9(10通り)である。<br>
一方、コンピュータ内部のデータは0,1(2通り)を使用した2進数で表現される。<br>
AP試験では各進数(2進数,8進数,16進数)間の基数変換に関する問いが頻出される。<br>
本記事では、**基数変換に関する重要事項**をまとめる。<br>
<br>

#### 基数変換の計算
- 2進数/8進数/16進数から10進数への変換<br>
 基数のべき乗に各桁をかけて加える  
  - ex)2進数から10進数への変換  
  `` 
 (1011)2 = 1*2^3 + 0*2^2 + 1*2^1 + 1*1^1 = 8+2+1 = 11
  ``  
  - ex)16進数から10進数への変換  
  ``
    (F5A)16 = 15*16^2 + 5*16^1 + 10*16^0 = 3930
  ``<br><br>
- 10進数から2進数/8進数/16進数への変換  
  (abc.de)nについて考える。
  - 整数部  
   ``a*n^2 + b*n^1 + c*n^0 = (a*n+b)*n + c``  
   nで割っていくと、順にc,b,aが求められる。  
   これを上位の桁から並べて、abcとなる。
  - 小数部  
   ``d*n^-1 + e*n^-2 = (d+e*n^-1)*n^-1``  
   nを掛けていくと、整数値として順にd,eが求められる。  
   これを小数第一位から順に並べて、deとなる。
  <br><br>
- 2進数,8進数,16進数への相互変換  
  - まずは2進数に変換する
  - 変換先の進数である2^nのnビットごとに区切って値を求める。  
  ``(576.2)8 = (101 111 110. 010)2 = (0001 0111 1110. 0100)2 = (17E.4)16``
<br><br>
### 補数表現
  - ある数の補数とは、ある数とその補数を足すとその桁数の最大値になる数、又は、ある数とその補数を足すとその桁数の最大値に1を加えた数になるような数のこと。
  - あらゆる進数に補数の概念は存在。「補数」、「元基数」の概念が存在。 
  - 基数の場合の合計数 : nのm乗  
  - 元気数の場合の合計数 : nのm乗-1  
  - メリット : 引き算を使用せずに足し算のみで負の数を表現できる点  
``6の補数は4 (10^1 = 10)　→10の補数``  
``171の補数は829 (10^3 = 1000)　→10の補数``  
``6の補数は3 (10^1-1 = 9)　→9の補数``   
``171の補数は828 (10^3-1 = 999)　→9の補数``  


即ち、**「1の補数」はビット反転した数、「2の補数」はビット反転したものに1加えた**数
